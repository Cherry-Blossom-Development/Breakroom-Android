3. Running & Debugging
======================

  Start Emulator (from Android Studio or command line):

    emulator -avd Pixel_7_API_34

  Check connected devices:

    adb devices

  Build debug APK:

    cd C:\Users\dalla\AndroidStudioProjects\MyFirstApp
    .\gradlew.bat assembleDebug

  Install APK to device/emulator:

    adb install app\build\outputs\apk\debug\app-debug.apk


4. Using Claude Code for Android Development
=============================================

  Start Claude inside your project:

    cd C:\Users\dalla\AndroidStudioProjects\MyFirstApp
    claude

  Claude understands:
    - Gradle files (build.gradle.kts, settings.gradle.kts)
    - Kotlin source code
    - AndroidManifest.xml
    - Project structure

  Example Prompts:

    Generate UI:
      "Create a Jetpack Compose screen with a login form,
       email + password validation, and Material 3 styling."

    Architecture:
      "Refactor this app to MVVM with ViewModel, StateFlow,
       and a repository layer."

    Debugging:
      "Analyze this crash log and explain why the app crashes
       when rotating the device."

    Gradle Fixes:
      "Fix Gradle build errors and update dependencies to
       Android Gradle Plugin 8+."

    API Integration:
      "Add Retrofit + Hilt + Kotlin Serialization for a REST API."


5. Best Practices
=================

  Do:
    - Ask Claude to modify existing files
    - Ask for diff-style changes
    - Ask for tests (JUnit + Compose UI tests)
    - Ask for performance improvements

  Avoid:
    - Letting Claude blindly upgrade all dependencies
    - Asking it to "rewrite everything" without constraints


6. Creating a New Project
=========================

  In Android Studio:
    1. File > New > New Project
    2. Choose Empty Activity (or Empty Compose Activity)
    3. Language: Kotlin
    4. Minimum SDK: API 24+
    5. Build configuration: Kotlin DSL (Recommended)

  Then open in Claude Code:

    cd C:\Users\dalla\AndroidStudioProjects\YourNewApp
    claude

Note on WSL
===========

  WSL with Ubuntu is installed on this machine but not configured as default.
  The Windows-native approach was chosen as it is simpler and fully functional.

  If WSL is needed in the future:
    1. Enable virtualization in BIOS (VT-x or AMD-V)
    2. Enable Virtual Machine Platform in Windows Features
    3. Run: wsl --set-default Ubuntu
